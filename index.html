Algorithm Design

분류 알고리즘 선택 근거

K-Means가 아닌 이유, 그리고 어떤 조합이 가장 신뢰도 높은 판정을 만드는지 설명합니다.

❌ K-Means를 쓰면 안 되는 이유

TRL은 지도학습(Supervised) 문제입니다

K-Means는 라벨 없이 군집을 찾는 비지도 학습입니다. TRL은 1~9의 순서가 있고, Low/Mid/High 라벨이 명확히 정의되어 있습니다. 또한 Mid가 82.6%로 편중된 상황에서 K-Means를 쓰면 모든 샘플이 Mid로 수렴합니다. High TRL(2.5%)은 아예 감지조차 못 합니다.

K-Means ❌ 부적합
비지도 클러스터링
TRL은 라벨이 있는 지도학습 문제. 군집 번호와 TRL 단계를 매핑할 방법이 없음. 클래스 불균형(Mid 82%)에서 전부 Mid로 수렴.
라벨 없음 불균형 취약 확률 출력 불가
Temperature Scaling ✅ 핵심
Calibration (확률 보정)
Softmax는 신뢰도를 과대평가합니다 (95% 확신인데 실제로는 70%). Temperature Scaling이 이를 보정해 실제 정확도와 신뢰도를 일치시킵니다. 모델 재학습 없이 파라미터 T 하나만 조정.
과신뢰 방지 재학습 불필요 튀는 값 감소
MC Dropout ✅ 핵심
Monte Carlo Dropout (불확실성)
추론 시 Dropout을 켜고 30회 반복 → 확률 분포의 분산 계산. 분산이 크면 "이 샘플은 애매함" → 재검토 필요 플래그 자동 설정. 튀는 값 방지에 가장 효과적.
불확실성 정량화 재검토 자동 감지 속도 30배 느림
Confidence Threshold ✅ 즉시 적용
거부 옵션 (Reject Option)
신뢰도 < 0.6이면 판정을 내리지 않고 "전문가 재검토 필요"로 표시. 구현 1줄, 즉시 적용 가능. 잘못된 High 판정 방지에 매우 효과적.
구현 1줄 오판 방지 일부 미판정 발생
Focal Loss ✅ 학습 적용
손실 함수 (Loss Function)
쉬운 샘플(Mid)의 가중치를 낮추고 어려운 샘플(High)에 집중. High TRL 2.5% 희소 클래스 성능을 향상. gamma=2.0이 최적.
High F1 향상 불균형 해소 gamma 조정 가능
Ensemble Voting △ 선택적
앙상블 (모델 3개 조합)
KLUE-RoBERTa + GPT-4o-mini FT + mDeBERTa 3개 모델 투표. 3개가 동의할 때만 High 판정 → 최고 안정성. 단, 모델 3개 유지 비용 발생.
최고 안정성 비용 3배 속도 3배 느림

# 계획서 텍스트 직접 입력
from kotrl_pipeline import predict_trl_from_proposal

result = predict_trl_from_proposal(
    proposal_text="""
    본 연구는 딥러닝 기반 반도체 결함 검출 시스템을 개발한다.
    실험실 환경에서 핵심 알고리즘을 검증하고,
    파일럿 라인 적용을 목표로 한다.
    """,
    tech_name="AI 검사 시스템",
    year=2024
)

print(result['level'])        # Low / Mid / High
print(result['trl_range'])    # TRL 4~6 (연구실 검증 / 시제품)
print(result['confidence'])   # 0.743
print(result['needs_review'])  # False (신뢰도 >= 0.6)

# MC Dropout으로 불확실성 추정
import torch
import numpy as np

def mc_predict(model, tokenizer, text, meta, n=30):
    # Dropout 활성화 상태로 N회 반복 추론
    model.train()  # Dropout 켜기
    probs_list = []

    with torch.no_grad():
        for _ in range(n):
            probs = predict_probs(model, tokenizer, [text], meta)
            probs_list.append(probs[0])

    probs_arr = np.array(probs_list)  # (30, 3)
    mean      = probs_arr.mean(axis=0)
    std       = probs_arr.std(axis=0)

    level_id    = mean.argmax()
    uncertainty = float(std.max())

    return {
        'level':       ID2LABEL[level_id],
        'confidence':  float(mean[level_id]),
        'uncertainty': uncertainty,
        'needs_review': uncertainty > 0.05 # 불확실하면 재검토
    }
